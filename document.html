📡 API Endpoints

1. POST /api/{key}/send

Recommended method for longer messages and better security

Endpoint
POST https://dms-api.mcboss.top/api/{your-api-key}/send
Request Body (JSON)
{
  "email": "user@example.com",
  "message": "Hello! This is my message to support.",
  "discord_id": "123456789012345678" // ← NEW: Optional Discord ID
}

2. GET /api/{key}/email-{email}/message-{message}

Simple GET method for basic implementations

Endpoint
GET https://dms-api.mcboss.top/api/{key}/email-{email}/message-{message}
Example
https://dms-api.mcboss.top/api/your-key/email-john@example.com/message-Need%20help%20with%20my%20order

3. NEW: GET with Discord ID

Enhanced GET method with Discord user identification

Endpoint
GET https://dms-api.mcboss.top/api/{key}/email-{email}/message-{message}/id-{discord-id}
Example
https://dms-api.mcboss.top/api/your-key/email-john@example.com/message-Need%20help/id-123456789012345678
🆕 Discord ID Enhancement:
• Discord ID parameter is optional in all endpoints
• Must be numeric (Discord user ID format)
• Automatically fetches and displays Discord username
• Fully backward compatible with existing implementations

🛡️ Security Features

🔒 Our Security Promise: We take security seriously. Your API key and messages are protected by multiple layers of military-grade security that prevents unauthorized access even with valid API keys.
🔐 NEW: Advanced Owner Permission Verification
Revolutionary security system that verifies API key owner permissions in real-time. Even if someone steals your API key, they can't abuse it without your admin permissions!

🆕 Owner Permission Verification

Our groundbreaking check_user_admin_permissions() function performs real-time verification:

✅ Multi-Layer Permission Checks:
• Verifies API key owner is actually in the server
• Confirms owner has Administrator permissions
• Handles server owner detection (automatic admin rights)
• Blocks requests if owner loses permissions
• Works on EVERY API request - no exceptions

🚨 Real-Time Security Enforcement

Before sending ANY message, our API now performs comprehensive security checks:

Security Check Sequence:
1. Validate API key exists
2. Check if owner is still in server
3. Verify owner has admin permissions
4. Validate URL authentication
5. Only then process the request
🛡️ Attack Prevention: Even with a valid API key, attackers CANNOT abuse the system because:
  • API checks the KEY OWNER's permissions (not the attacker's)
  • Only legitimate admins who created the key can use it
  • Permission verification happens on every single request
  • Lost admin permissions = API key stops working instantly

Common Issues & Solutions

🚫 "Request must come from authorized URL"
This error means your request is not coming from the URL you registered. Make sure:
  • You're testing from the exact URL you registered
  • Your URL includes the correct protocol (http/https)
  • You haven't changed domains since registration
🤖 "Bot is not in the server"
Our Discord bot needs to be in your server. Use the invite link and make sure the bot has permission to:
  • Send Messages
  • Embed Links
  • Read Message History
📧 "Invalid email format"
Make sure the email address follows the standard format: name@domain.com
